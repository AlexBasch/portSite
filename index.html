This project was to demonstrate our knowledge of creating and consuming an API using a back- and frontend framework technology of our choice as well as making a responsive web application.

In the backend I used php and Slim as my framework to create my API.

In the front I used AngularJS to consume my API along with Bootstrap for the responsiveness.

Java

This project was to demonstrate our knowledge of creating a Java application as well as a basic GUI for either a pizza ordering system or and airport booking system. I chose to go with the airport system, requiring the following set of criteria to be filled.

  • Work using a fully functional GUI.
  • Have a search functionality for previously booked tickets.
  • Make new bookings, where the user picks type and quality of booking as well as the desired plane to book on.
  • Calculate costs of all variables, working in membership discount.
  • Save states.

For this project I needed to sharpen up on my java, needing to learn how to create and layout a GUI using only java as well as solve the problem of how to create the best possible abstract classes for all the modules I needed. These abstractions would send back values depending on the selected criteria which was used to calculate the final fee.

Android

For this project we were give 2 android technologies and had to create an app that used both. I received the camera and content providers.

For this project the task set before us was simply to use the above mentioned technologies in any way we wanted to as long as both technologies were used together in some way or another.

My app was a basic contact creator, using the camera to take a profile picture and then adding it to the contacts using the contacts contracts content provider. This was done using the launch camera action instead of the full camera api as well as the contact contracts push function without the full raw contact contracts action.
